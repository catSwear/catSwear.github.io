原型链和继承

原型链

基本概念：

构造函数：类似蓝图。是function.

实例对象：用构造函数new出来的，类似通过蓝图制作的具体产品。是object.

原型对象：是object.

它们的关系：

构造函数.prototype -> 原型对象

原型对象.constructor -> 构造函数

实例对象.__ proto__ -> 原型对象

原型对象也是object，它也有__ proto__的属性，于是

构造函数.prototype.__ proto__ -> Object.prototype

原型链：

实例对象.__ proto__ -> 原型对象

原型对象.__ proto__ -> Object.prototype

Object.prototype.__ proto__ -> null

继承：

基本概念：

子类：作为一个类，其实就是构造函数，作为将要继承方法的构造函数，是fucntion

父类：作为被继承方法的构造函数,是function

原型的继承：子类可以访问到父类的属性和方法

具体操作：

让父类的属性和方法出现在子类实例的原型链上

让子类的prototype属性指向父类的实例对象

此时，子类的原型对象成了父类的一个实例对象，实例对象中不存在constructor属性，需要自己重定向到子类中。保持继承后原型链的完整。

之后，子类创建的实例对象就能通过新的原型链访问到父类的属性和方法了。

如何处理npm与nodejs版本冲突

方法：

使用npm install npm 也会提示更新版本

卸载nodejs 去官网下载最新版重新安装

但是要卸载环境变量和所有npm的信息

一、环境变量：

1.用户环境变量和系统变量都要删除

二、npm残留文件

使用一段时间会出现npm和npm_cache两个文件夹。AppData是隐藏文件夹要显示出来，删掉之后。再卸载nodejs就卸载干净了
